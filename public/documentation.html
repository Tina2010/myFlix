myFlix - Api Documentation

The following page provides all necessary context to understand how to API works and how it is possible to run the API.

Description:

The myFlix API offers a way to provide users with access to information about different movies, directors, and genres. You can take a first look at this APIs work in my application: myFlix.

URL Endpoints

  • Movies

    Request URL HTTP Method Request Data Format Response Data Body Format
    Get a list of all movies. /index.html GET None A JSON Object holding data about all movies.
    Get details (description, genre, director, image URL) of a single movie. /top10movies/:title GET None A JSON Object holding data about a single movie.
    Get genre (name and description) of a single movie. /top10movies/genre/:name GET None A JSON Object holding data about a genre of a single movie.
    Get all details of a director. /top10movies/director/:name GET None A JSON Object holding data about a single director.

  • Users

    Request URL HTTP Method Request Data Format Response Data Body Format
    Create information of a new user (username). /users POST A JSON Object holding data about the user to add, structured like:
    {
    username: "SomeName"
    }     		A JSON Object holding data about the user that was added, including an ID:
    {
    id: 1,
    username: "SomeName"
    }
    Update information of a user (username). /users/:username PUT A JSON Object holding the username of the user, structured like:
    {
    username: "SomeName"
    }     		A JSON Object holding the username of the user, structured like:
    {
    username: "SomeName"
    }
    Remove information of a user (username). /users/:id DELETE A JSON Object holding data about the user to add, structured like:
    {
    id: ""
    }     		A status response telling the user, that he deregistered.

  • Favorite Movies

    Request URL HTTP Method Request Data Format Response Data Body Format
    Add movies to your favorites list. /index.html/:title/:favorite PUT A JSON Object holding data of a movie in addition to a checked favorite option, structured like:
    {
    title: "movie1"
    , favorite: "yes"
    }     		A status response telling the user, that his movie was added to his favorites list.
    Remove movies from your favorites list. /index.html/:title/:favorite PUT A JSON Object holding data of a movie in addition to a unchecked favorite option, structured like:
    {
    title: "movie1"
    , favorite: "no"
    }     		A status response telling the user, that his movie was removed from his favorites list.
